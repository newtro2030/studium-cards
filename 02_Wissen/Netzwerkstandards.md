### Warum sind Standards wichtig?

Netzwerkstandards ermöglichen **Interoperabilität** zwischen Geräten und Software verschiedener Hersteller.

Standards definieren:
- Format der Datenpakete
- Verhalten von Protokollen
- Schnittstellen

Sie ermöglichen Austauschbarkeit, Herstellerunabhängigkeit und weltweite Skalierbarkeit.

### Standardisierungsorganisationen

| Organisation | Zuständigkeit | Beispiele |
|---|---|---|
| **ISO** | Internationale Normung | OSI-Referenzmodell |
| **ITU** | Telekommunikation (Teil der UNO) | Telefonie, Mobilfunk |
| **IEEE** | Netzwerktechnik | Ethernet (802.3), WLAN (802.11), Bluetooth (802.15) |
| **IETF** | Internetprotokolle | TCP/IP, HTTP, DNS, IPv6 |
| **W3C** | Webtechnologien | HTML, CSS, XML |

### Was ist ein RFC?

- **Request for Comments** – Veröffentlichungen der IETF
- Nummeriert, öffentlich zugänglich
- Einmal veröffentlicht **niemals mehr geändert** → neue Version = neue Nummer
- Beispiele: RFC 791 = IPv4, RFC 2460 = IPv6

> **Merksatz:** Netzwerkstandards sind das Fundament globaler Kommunikation – ohne sie würde kein Gerät mit einem anderen sprechen.
